# PaperBox 重建架构

## 产品定位

一个面向个人研究和知识工作的本地桌面应用，核心价值是把“资料归档、内容检索、笔记沉淀、AI 分析”放到同一个闭环中。

## 目标能力

1. 本地文献库管理
2. 多格式文档导入与内容提取
3. 结构化元数据管理
4. 全文搜索与标签筛选
5. 论文笔记与知识分组
6. 基于选中文献上下文的 AI 对话与摘要

## 推荐模块拆分

### Main Process

- `window`: 窗口生命周期
- `database`: SQLite 初始化、查询封装、迁移
- `library`: 文献导入、文件复制、哈希去重
- `parser`: PDF/TXT/DOCX 内容提取
- `search`: FTS 更新与搜索查询
- `ai`: 预设配置、流式对话、摘要生成
- `export`: Markdown 导出、摘要导出

### Preload

- 暴露有限 IPC API
- 统一返回结构与错误处理

### Renderer

- `layout`: 三栏或四栏布局
- `library`: 列表、筛选、排序、批量操作
- `paper-detail`: 元数据、摘要、标签、笔记入口
- `notes`: 树形笔记与编辑器
- `chat`: 多轮对话、流式输出、上下文选择
- `settings`: 模型、目录、导入策略、检索设置

## 为什么保留 Electron

1. 原产品就是桌面应用，用户心智一致
2. 本地文件访问、SQLite、文档解析都更顺手
3. 后续如果需要自动监听导入目录、离线模式、Ollama 集成，桌面端成本更低
